TaxDC: A Taxonomy of Non-Deterministic Concurrency Bugs in Datacenter Distributed Systems

Aug 29, 2016 | code

“TaxDC: A Taxonomy of Non-Deterministic Concurrency Bugs in Datacenter Distributed Systems”是国际顶级系统结构会议ASPLOS2016录用的文章，这次会议于2016年4月在美国亚特兰大召开。文章对分布式系统上的并发错(DC-bugs)进行了系统研究，并对DC-bugs的特点进行了系统的分类总结。

自从并发软件出现，并发错误就影随其形。不幸的是，不同于其他的错误，并发错误的发生具有随机性。其触发条件不仅和输入有关，还和运行环境有关，如系统调度、消息传递时机。现有大量的工作对并发错误进行研究，一般的并发错误分为：数据竞争、顺序违例、原子违例和死锁。

然而，传统的研究只针对本地单机上的并发软件（如多线程程序）上的并发错误，而对如今磅礴发展的分布式系统（如Hadoop MapReduce）上的并发错误的研究比较少。而对分布式系统上的并发错误进行总结性研究还没有。因此，作者对分布式系统上的并发错误进行了系统的分类研究。

作者从四个非常著名的云环境下数据中心分布式系统，包括Cassandra、Hadoop MapReduce、HBase和ZooKeeper，上收集104个DC-bugs进行研究。作者研究DC-bugs特点包括：触发的时间条件和输入条件、错误分类和修复策略。

通过对104个DC-bugs研究，作者有如下发现：

  1. 对于出现的DC-bugs，缺少有效的测试、验证和分析工具。
  2. 现实中DC-bugs非常难以查找，因为它们涉及到多种并发执行方式和多种复杂协议。
  3. 63%的DC-bugs伴随于硬件错误，如机器重启或崩溃、网络异常和磁盘错误。当这些硬件错误出现时，协议会创建更多非确定性时间，并与其它机器上的事件并发执行。
  4. 47%的DC-bugs会导致隐秘错误，而这些错误难以重现或调试。
  5. 60%的DC-bugs由单个异常消息触发，其会导致顺序违例和原子违例。
  6. 53%的DC-bugs会导致明显的本地或全局错误。
  7. 大部分DC-bugs只由少量的方案来修复。30%通过阻止触发时间条件的出现，另外有40%通过延迟或无视异常消息，或接收这些消息但不引入新的处理逻辑。
